## 1. Что такое SQL? Для чего он применяется?

SQL (Structured Query Language) - это язык структурированных запросов, стандартный язык для управления и манипулирования реляционными базами данных.

SQL применяется для:
- Создания и модификации структуры базы данных (таблицы, индексы, представления)
- Хранения и извлечения данных из базы данных
- Обновления данных в базе данных
- Удаления данных из базы данных
- Управления доступом к базе данных

SQL является декларативным языком, то есть мы указываем, что хотим получить, а не как именно получить это. Он универсален и может использоваться с различными реляционными СУБД, такими как MySQL, PostgreSQL, Oracle, SQL Server и другими.

---

## 2. В чем отличие между DDL, DML, DCL и TCL? Какие операции входят в каждую из групп?

**DDL (Data Definition Language)** - язык определения данных:
- CREATE - создание объектов базы данных (таблиц, представлений, индексов)
- ALTER - изменение структуры существующих объектов
- DROP - удаление объектов базы данных
- TRUNCATE - удаление всех записей из таблицы

**DML (Data Manipulation Language)** - язык манипуляции данными:
- SELECT - выборка данных из базы данных
- INSERT - вставка новых данных
- UPDATE - обновление существующих данных
- DELETE - удаление данных

**DCL (Data Control Language)** - язык контроля данных:
- GRANT - предоставление разрешений пользователям
- REVOKE - удаление разрешений у пользователей

**TCL (Transaction Control Language)** - язык управления транзакциями:
- COMMIT - фиксация транзакции
- ROLLBACK - откат транзакции
- SAVEPOINT - создание контрольной точки для частичного отката

## 3. Перечислите все типы связей таблиц в SQL.

**1. Один к одному (One-to-One)**:
- Каждая запись в таблице A связана с одной записью в таблице B
- Пример: отношение между сотрудником и паспортом

**2. Один ко многим (One-to-Many)**:
- Одна запись в таблице A может быть связана с несколькими записями в таблице B
- Пример: отношение между клиентом и заказами

**3. Многие ко многим (Many-to-Many)**:
- Несколько записей в таблице A могут быть связаны с несколькими записями в таблице B
- Обычно реализуется через промежуточную таблицу
- Пример: отношение между студентами и курсами

**4. Селф-связь (Self-join)**:
- Таблица связана сама с собой
- Пример: отношение между сотрудником и его менеджером

## 4. Что такое СУБД?

СУБД (Система Управления Базами Данных) - это программное обеспечение, которое позволяет создавать, использовать и администрировать базы данных.

Функции СУБД:
- Хранение и извлечение данных
- Управление транзакциями
- Обеспечение безопасности данных
- Контроль целостности данных
- Обеспечение многопользовательского доступа
- Обеспечение отказоустойчивости

Примеры СУБД: MySQL, PostgreSQL, Oracle, Microsoft SQL Server, SQLite.

## 5. В чем разница между базой данных и схемой?

**База данных** - это организованная коллекция структурированных данных, которая хранится и доступна электронным способом. Она представляет собой полное хранилище данных, включающее таблицы, представления, индексы и другие объекты.

**Схема** - это логическая структура базы данных, которая определяет, как данные организованы и связаны друг с другом. Схема описывает структуру таблиц, их столбцы, типы данных, ограничения и связи между таблицами.

В контексте конкретной СУБД:
- Схема может быть отдельным пространством имен в базе данных (например, в PostgreSQL)
- База данных содержит одну или несколько схем
- Схема определяет логическую структуру данных

## 6. SQL vs NoSQL

**SQL (реляционные базы данных):**
- Используют строго определенную схему
- Поддерживают ACID-транзакции
- Используют язык SQL для запросов
- Масштабируются вертикально
- Подходят для сложных запросов с JOIN
- Примеры: MySQL, PostgreSQL, Oracle

**NoSQL (нереляционные базы данных):**
- Гибкая или отсутствующая схема
- Обычно обеспечивают высокую доступность
- Используют различные языки запросов
- Масштабируются горизонтально
- Подходят для больших объемов данных
- Примеры: MongoDB (документо-ориентированная), Redis (ключ-значение), Cassandra (колоночная)

## 7. Что такое нормализация и каковы ее преимущества?

Нормализация - это процесс организации данных в базе данных для уменьшения избыточности и зависимости. Это достигается путем разделения больших таблиц на более мелкие и определения отношений между ними.

Преимущества нормализации:
- Уменьшение избыточности данных
- Улучшение согласованности данных
- Упрощение обновления данных
- Снижение вероятности ошибок при вводе данных
- Экономия дискового пространства

---

## 8. Какие уровни нормализации есть в SQL?

**1NF (Первая нормальная форма)**:
- Каждый столбец содержит только атомарные (неделимые) значения
- Каждая запись уникальна

**2NF (Вторая нормальная форма)**:
- Должна быть в 1NF
- Все неключевые атрибуты полностью зависят от первичного ключа

**3NF (Третья нормальная форма)**:
- Должна быть в 2NF
- Нет транзитивных зависимостей (неключевые атрибуты не зависят от других неключевых атрибутов)

**BCNF (Нормальная форма Бойса-Кодда)**:
- Расширенная версия 3NF
- Каждая детерминанта является кандидатом на первичный ключ

**4NF (Четвертая нормальная форма)**:
- Должна быть в BCNF
- Отсутствие многозначных зависимостей

**5NF (Пятая нормальная форма)**:
- Должна быть в 4NF
- Отсутствие зависимостей соединения

## 9. Денормализация: определение

Денормализация - это процесс преднамеренного нарушения нормальных форм в целях улучшения производительности чтения данных. При денормализации вносятся избыточные данные или отношения для упрощения запросов.

Преимущества:
- Улучшение производительности запросов
- Упрощение структуры запросов
- Снижение количества соединений (JOIN)

Недостатки:
- Увеличение избыточности данных
- Потенциальные проблемы с целостностью данных
- Сложности при обновлении данных
- Увеличение использования дискового пространства

## 10. В чем разница между типом данных `CHAR` и `VARCHAR` в SQL?

**CHAR (Character)**:
- Фиксированная длина
- Занимает постоянное количество места в памяти
- Если значение короче, чем объявленная длина, дополнится пробелами
- Быстрее для операций, но может быть неэффективным по памяти
- Пример: CHAR(10) всегда занимает 10 байтов, независимо от длины значения

**VARCHAR (Variable Character)**:
- Переменная длина
- Занимает только столько места, сколько необходимо для фактических данных
- Не дополняет значения пробелами
- Более эффективно по памяти, но может быть медленнее для некоторых операций
- Пример: VARCHAR(10) может содержать от 0 до 10 символов

## 11. Разница дат в SQL

Для вычисления разницы между датами в SQL используются различные функции в зависимости от СУБД:

**MySQL**:
- DATEDIFF(date1, date2) - разница в днях
- TIMESTAMPDIFF(unit, datetime1, datetime2) - разница в указанной единице (SECOND, MINUTE, HOUR, DAY, MONTH, YEAR)

**PostgreSQL**:
- date1 - date2 - разница в днях
- age(timestamp1, timestamp2) - разница в годах, месяцах и днях

**SQL Server**:
- DATEDIFF(interval, date1, date2) - разница в указанном интервале (DAY, MONTH, YEAR, HOUR и др.)

**Пример**:
```sql
SELECT DATEDIFF(CURDATE(), birth_date) AS days_since_birth FROM users;
```

## 12. Что такое первичный ключ (Primary key) в таблице?

Первичный ключ (Primary Key) - это столбец или комбинация столбцов, которая уникально идентифицирует каждую запись в таблице.

Характеристики первичного ключа:
- Должен быть уникальным для каждой записи
- Не может содержать NULL-значения
- Может быть только один в таблице
- Обычно является индексированным для быстрого поиска
- Используется для установления связей с другими таблицами через внешние ключи

---

## 13. Объясните разницу между первичным и внешним ключом.

**Первичный ключ**:
- Уникально идентифицирует записи в таблице
- Не может содержать NULL-значения
- Только одна первичная ключевая колонка на таблицу (или комбинация колонок)
- Автоматически создает уникальный индекс

**Внешний ключ**:
- Ссылается на первичный ключ другой таблицы (или той же таблицы)
- Может содержать NULL-значения
- Может быть несколько внешних ключей в одной таблице
- Поддерживает целостность ссылок между таблицами
- Может иметь повторяющиеся значения

---

## 14. Что такое уникальный ключ (Unique key)?

Уникальный ключ - это ограничение, которое гарантирует, что все значения в столбце или наборе столбцов уникальны.

Отличия от первичного ключа:
- Может содержать NULL-значения (обычно только одно NULL)
- Может быть несколько уникальных ключей в одной таблице
- Может содержать NULL-значения
- Не обязательно быть кластеризованным индексом

---

## 15. Временные таблицы в SQL

Временные таблицы существуют только в рамках текущего сеанса или транзакции и автоматически удаляются после завершения сеанса.

**Создание временной таблицы**:
```sql
CREATE TEMPORARY TABLE temp_table (
    id INT,
    name VARCHAR(50)
);
```

Преимущества:
- Полезны для промежуточных вычислений
- Не влияют на постоянные данные
- Автоматически удаляются при завершении сеанса
- Изолированы для каждого пользователя

## 16. Ограничения столбцов в SQL

**PRIMARY KEY** - уникальный идентификатор, не может быть NULL
**FOREIGN KEY** - ограничение ссылочной целостности
**UNIQUE** - уникальные значения в столбце
**NOT NULL** - столбец не может содержать NULL
**CHECK** - проверка условия для значений
**DEFAULT** - значение по умолчанию для столбца

---

## 17. Как изменить тип данных столбца?

**PostgreSQL**:
```sql
ALTER TABLE table_name ALTER COLUMN column_name TYPE new_data_type;
```

**MySQL**:
```sql
ALTER TABLE table_name MODIFY COLUMN column_name new_data_type;
```

**SQL Server**:
```sql
ALTER TABLE table_name ALTER COLUMN column_name new_data_type;
```

Важно: при изменении типа данных могут возникнуть ошибки, если существующие данные несовместимы с новым типом.

## 18. Что такое индексы в SQL? Зачем они нужны?

Индекс - это структура данных, которая улучшает скорость извлечения данных из таблицы. Индекс содержит ключи, полученные из одной или нескольких колонок таблицы, и указатели на соответствующие строки таблицы.

Почему нужны индексы:
- Ускорение поиска данных
- Ускорение сортировки
- Улучшение производительности JOIN-операций
- Быстрая проверка ограничений уникальности

Недостатки:
- Замедление операций INSERT, UPDATE, DELETE
- Дополнительное использование дискового пространства

## 19. Виды отказов в СУБД

**Отказы транзакций**:
- Процесс завершается аварийно до завершения транзакции
- Обычно вызваны ошибками в приложении

**Отказы системы**:
- Сбой системы (например, сбой питания)
- Транзакции не записываются на диск, теряются в памяти

**Отказы носителей хранения**:
- Физическое повреждение диска или носителя
- Потеря данных, требует восстановления из резервных копий

## 20. Какие виды индексов существуют в PostgreSQL?

**B-Tree** - по умолчанию, для равенства и упорядочения
**Hash** - для операций равенства
**GiST** - обобщенная структура поиска, поддерживает различные алгоритмы
**SP-GiST** - для нетривиальных структур данных
**GIN** - для массивов, полнотекстового поиска
**BRIN** - блочные диапазонные индексы, для больших таблиц

## 21. Почему B-Tree является индексом по умолчанию?

B-Tree (сбалансированное дерево) является индексом по умолчанию, потому что:
- Эффективен для операций равенства и упорядочения
- Поддерживает как точные совпадения, так и диапазонные запросы
- Хорошо справляется с вставками и удалениями
- Поддерживает сортировку данных
- Всегда остается сбалансированным, обеспечивая стабильную производительность

## 22. Что такое индексация полнотекстового поиска в SQL и как она работает?

Полнотекстовый поиск позволяет искать слова или фразы внутри текстовых данных. Он создает индекс, который позволяет быстро находить документы, содержащие заданные слова.

Как работает:
- Текст анализируется и разбивается на отдельные слова (токены)
- Применяются правила нормализации (стемминг, удаление стоп-слов)
- Создается инвертированный индекс с соответствиями слов и документов
- При поиске быстро находятся документы, содержащие искомые слова

В PostgreSQL используется тип данных tsvector и tsquery, а также оператор @@ для поиска.

## 23. Что такое представление (VIEW) в SQL и как оно используется?

Представление (VIEW) - это виртуальная таблица, результат которой определяется SQL-запросом. Представление не хранит данные физически, а отображает данные из одной или нескольких таблиц.

Создание представления:
```sql
CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;
```

Преимущества:
- Упрощение сложных запросов
- Добавление уровня безопасности (ограничение доступа к данным)
- Переиспользование запросов
- Абстрагирование сложной логики

## 24. В чем разница между операторами `DELETE` и `TRUNCATE`?

**DELETE**:
- Удаляет строки по одной
- Можно использовать с WHERE для условного удаления
- Можно отменить с помощью ROLLBACK
- Выполняет триггеры
- Медленнее

**TRUNCATE**:
- Удаляет все строки из таблицы
- Быстрее, так как не удаляет по одной
- Не может быть отменен в транзакционной СУБД (если используется внутри транзакции)
- Не вызывает триггеры
- Сбрасывает счетчики автоинкремента

## 25. В чём разница между операторами `UNION` и `UNION ALL`?

**UNION**:
- Объединяет результаты двух или более SELECT-запросов
- Удаляет дубликаты из результата
- Требует больше ресурсов из-за операции удаления дубликатов

**UNION ALL**:
- Объединяет результаты двух или более SELECT-запросов
- Сохраняет все дубликаты
- Быстрее, так как не удаляет дубликаты

Пример:
```sql
SELECT name FROM table1
UNION
SELECT name FROM table2;

SELECT name FROM table1
UNION ALL
SELECT name FROM table2;
```

## 26. Объясните функцию `COALESCE` в SQL

Функция COALESCE возвращает первое ненулевое значение из списка аргументов. Она используется для замены NULL-значений альтернативными значениями.

Синтаксис:
```sql
COALESCE(expression1, expression2, ..., expression_n)
```

Пример:
```sql
SELECT COALESCE(phone, 'Нет телефона') AS contact_info FROM users;
```

Если phone равен NULL, то будет возвращено 'Нет телефона'.

## 27. Объясните оператор `LIKE` и его использование

Оператор LIKE используется для поиска по шаблону в строковых данных. Он часто используется с метасимволами:

**%** - любое количество символов (включая ноль)
**_** - один символ

Примеры:
```sql
-- Найти имена, начинающиеся с 'A'
SELECT * FROM users WHERE name LIKE 'A%';

-- Найти имена, где второй символ 'o'
SELECT * FROM users WHERE name LIKE '_o%';

-- Найти имена, содержащие 'an'
SELECT * FROM users WHERE name LIKE '%an%';
```

## 28. Зачем нужен CASE в SQL

CASE - это условное выражение, которое позволяет выполнять разные действия в зависимости от условий. Это аналог операторов if-then-else в других языках программирования.

Синтаксис:
```sql
CASE
    WHEN condition1 THEN result1
    WHEN condition2 THEN result2
    ELSE result3
END
```

Пример:
```sql
SELECT name,
    CASE
        WHEN age < 18 THEN 'Несовершеннолетний'
        WHEN age BETWEEN 18 AND 65 THEN 'Взрослый'
        ELSE 'Пенсионер'
    END AS category
FROM users;
```

## 29. Поиск дублирующихся строк

Для поиска дубликатов можно использовать GROUP BY с HAVING:

```sql
SELECT column1, column2, COUNT(*)
FROM table_name
GROUP BY column1, column2
HAVING COUNT(*) > 1;
```

Или с помощью оконных функций:
```sql
SELECT *
FROM (
    SELECT *, 
           ROW_NUMBER() OVER (PARTITION BY column1, column2 ORDER BY id) AS rn
    FROM table_name
) t
WHERE rn > 1;
```

## 30. Для чего используется оператор `GROUP BY`?

GROUP BY используется для группировки строк, которые имеют одинаковые значения в указанных столбцах. Обычно используется с агрегатными функциями (COUNT, MAX, MIN, SUM, AVG).

Пример:
```sql
SELECT department, COUNT(*) as employee_count
FROM employees
GROUP BY department;
```

Этот запрос группирует сотрудников по отделам и подсчитывает количество сотрудников в каждом отделе.

## 31. Пагинация в SQL

Для пагинации используются LIMIT и OFFSET (в MySQL, PostgreSQL) или TOP и ROW_NUMBER() (в SQL Server).

MySQL/PostgreSQL:
```sql
SELECT * FROM table_name
LIMIT page_size OFFSET (page_number - 1) * page_size;
```

Пример для получения 10 записей на странице, 2-я страница:
```sql
SELECT * FROM products
LIMIT 10 OFFSET 10;
```

## 32. Что такое `ORDER BY` и как оно используется?

ORDER BY используется для сортировки результата запроса по одному или нескольким столбцам.

ASC - по возрастанию (по умолчанию)
DESC - по убыванию

Пример:
```sql
SELECT * FROM employees
ORDER BY salary DESC, name ASC;
```

Этот запрос сортирует сотрудников по зарплате по убыванию, а при равных зарплатах - по имени по возрастанию.

## 33. Для чего нужен оператор `UNION`?

UNION используется для объединения результатов двух или более SELECT-запросов в один результат. UNION автоматически удаляет дубликаты из результата.

- Количество столбцов в SELECT должно быть одинаковым
- Соответствующие столбцы должны иметь совместимые типы данных
- Столбцы в результирующем наборе будут иметь имена из первого SELECT

## 36. В чём разница между операторами `HAVING` и `WHERE`?

**WHERE**:
- Фильтрует строки до группировки
- Не может использоваться с агрегатными функциями
- Применяется до GROUP BY

**HAVING**:
- Фильтрует группы после группировки
- Может использоваться с агрегатными функциями
- Применяется после GROUP BY

Пример:
```sql
SELECT department, COUNT(*) as emp_count
FROM employees
WHERE salary > 50000
GROUP BY department
HAVING COUNT(*) > 5;
```

SELECT AVG(column_name) AS average_value

SELECT AVG(salary) AS average_salary

## 38. Агрегатные и скалярные функции

**Агрегатные функции**:
- Работают с набором значений и возвращают одно значение
- Примеры: AVG, COUNT, SUM, MAX, MIN
- Используются с GROUP BY

**Скалярные функции**:
- Работают с одним значением и возвращают одно значение
- Примеры: UPPER, LOWER, LENGTH, ROUND, SUBSTRING
- Применяются к каждой строке отдельно

## 39. Что такое `JOIN` в SQL? Какие виды `JOIN`'ов вы знаете?

JOIN используется для объединения строк из двух или более таблиц на основе связанного столбца между ними.

**INNER JOIN** - возвращает только совпадающие строки
**LEFT JOIN (LEFT OUTER JOIN)** - все строки из левой таблицы и совпадающие из правой
**RIGHT JOIN (RIGHT OUTER JOIN)** - все строки из правой таблицы и совпадающие из левой
**FULL OUTER JOIN** - все строки из обеих таблиц
**CROSS JOIN** - декартово произведение
**SELF JOIN** - таблица соединяется сама с собой

---

## 40. Чем отличается `INNER JOIN` от `LEFT JOIN`?

**INNER JOIN**:
- Возвращает только те строки, которые имеют совпадения в обеих таблицах
- Если нет совпадений, строка не включается в результат

**LEFT JOIN**:
- Возвращает все строки из левой таблицы
- Возвращает совпадающие строки из правой таблицы
- Если нет совпадений в правой таблице, возвращаются NULL для правых столбцов

## 41. Задача на JOIN

Задание: Есть две таблицы - employees (id, name, department_id) и departments (id, dept_name). Найти всех сотрудников с их отделами.

Решение:
```sql
SELECT e.name, d.dept_name
FROM employees e
INNER JOIN departments d ON e.department_id = d.id;
```

## 42. SELF JOIN

SELF JOIN - это соединение таблицы самой с собой. Используется, когда нужно сравнить строки внутри одной таблицы.

Пример: Найти сотрудников и их менеджеров (если есть):
```sql
SELECT e.name AS employee_name, m.name AS manager_name
FROM employees e
LEFT JOIN employees m ON e.manager_id = m.id;
```

## 43. CROSS и NATURAL JOIN

**CROSS JOIN**:
- Возвращает декартово произведение двух таблиц
- Каждая строка из первой таблицы соединяется с каждой строкой из второй таблицы
```sql
SELECT * FROM table1 CROSS JOIN table2;
```

**NATURAL JOIN**:
- Автоматически соединяет таблицы по столбцам с одинаковыми именами
- Не рекомендуется использовать, так как может привести к неожиданным результатам
```sql
SELECT * FROM table1 NATURAL JOIN table2;
```

## 44. Можете ли вы раскрыть смысл всех букв в аббревиатуре ACID?

ACID - это сокращение, описывающее набор свойств транзакций в базах данных:

**A (Atomicity)** - Атомарность: транзакция выполняется полностью или не выполняется вовсе
**C (Consistency)** - Согласованность: транзакция переводит базу данных из одного согласованного состояния в другое
**I (Isolation)** - Изолированность: транзакции изолированы друг от друга
**D (Durability)** - Устойчивость: зафиксированные изменения сохраняются даже при сбоях системы

---

## 45. Что такое транзакция в SQL и какими свойствами она должна обладать?

Транзакция - это последовательность операций, рассматриваемая как единое целое. Транзакция должна обладать свойствами ACID:

- Атомарность (Atomicity): либо все операции в транзакции выполняются, либо ни одна
- Согласованность (Consistency): транзакция сохраняет целостность данных
- Изолированность (Isolation): транзакции не влияют друг на друга
- Устойчивость (Durability): результаты выполненной транзакции сохраняются навсегда

---

## 46. Транзакции и блокировки

Блокировки используются для обеспечения согласованности данных при параллельном доступе к базе данных:

- **Разделяемые блокировки (Shared Locks)**: разрешают чтение, но не изменение данных
- **Исключительные блокировки (Exclusive Locks)**: запрещают другим транзакциям читать или изменять данные
- **Уровни изоляции**: READ UNCOMMITTED, READ COMMITTED, REPEATABLE READ, SERIALIZABLE

---

## 47. В чём разница между `COMMIT` и `SAVEPOINT`?

**COMMIT**:
- Сохраняет все изменения, сделанные в транзакции
- Делает изменения постоянными
- Завершает транзакцию

**SAVEPOINT**:
- Создает точку сохранения внутри транзакции
- Позволяет откатиться к этой точке, не завершая всю транзакцию
- Используется с ROLLBACK TO savepoint_name

---

## 48. Что такое грязное чтение в SQL базах данных?

Грязное чтение (Dirty Read) - это ситуация, когда транзакция читает данные, измененные другой транзакцией, которая еще не была зафиксирована (COMMIT).

Пример: 
- Транзакция A изменяет строку, но не фиксирует изменения
- Транзакция B читает измененную строку
- Транзакция A откатывается
- Транзакция B прочитала "грязные" данные

Это может быть предотвращено установкой уровня изоляции READ COMMITTED или выше.

---

## 49. Что такое хранимая процедура?

Хранимая процедура - это набор SQL-инструкций, который хранится в базе данных и может быть вызван по имени. Она может принимать параметры, выполнять сложную логику и возвращать результаты.

Преимущества:
- Повторное использование кода
- Улучшение производительности
- Централизованное управление логикой
- Повышение безопасности

Пример:
```sql
CREATE PROCEDURE GetEmployeeById(@id INT)
AS
BEGIN
    SELECT * FROM employees WHERE emp_id = @id;
END
```

## 50. Оконные функции и хранимые процедуры

**Оконные функции**:
- Выполняют вычисления над набором строк, связанных с текущей строкой
- Не сокращают количество строк, в отличие от агрегатных функций с GROUP BY
- Пример: ROW_NUMBER(), RANK(), LAG(), LEAD(), SUM() OVER()

**Хранимые процедуры**:
- Могут содержать оконные функции
- Позволяют инкапсулировать сложную бизнес-логику
- Могут возвращать результаты с использованием оконных функций

---

## 51. Что такое триггеры в SQL?

Триггер - это специальный тип хранимой процедуры, который автоматически выполняется при возникновении определенного события в базе данных (INSERT, UPDATE, DELETE).

Типы триггеров:
- DML триггеры: срабатывают при DML-операциях
- DDL триггеры: срабатывают при DDL-операциях
- LOGON триггеры: срабатывают при подключении к серверу

Преимущества:
- Поддержание целостности данных
- Автоматическое выполнение бизнес-логики
- Аудит изменений

## 52. Что такое подзапрос в SQL?

Подзапрос - это SELECT-запрос, вложенный внутрь другого SQL-запроса. Подзапрос может использоваться в:
- Заголовке SELECT
- Предложении FROM
- Предложении WHERE
- Предложении HAVING

Подзапрос должен быть заключен в скобки.

## 53. Типы подзапросов в SQL

**Коррелированные подзапросы**:
- Зависят от внешнего запроса
- Выполняются для каждой строки внешнего запроса

**Некоррелированные подзапросы**:
- Не зависят от внешнего запроса
- Выполняются один раз

**Одиночный подзапрос**:
- Возвращает одно значение

**Множественный подзапрос**:
- Возвращает несколько значений

## 56. Что делает команда `EXPLAIN ANALYZE` в SQL?

- Выполняет запрос и показывает план
- Отображает реальное время выполнения
- Показывает количество обработанных строк
- Показывает время, затраченное на каждый этап

## 59. Что такое `UNION`, `MINUS` и `INTERSECT` команды?

**UNION**:
- Объединяет результаты двух запросов
- Удаляет дубликаты

**UNION ALL**:
- Объединяет результаты двух запросов
- Сохраняет дубликаты

**INTERSECT**:
- Возвращает только строки, присутствующие в обоих результатах
- Удаляет дубликаты

**MINUS (EXCEPT в SQL Server)**:
- Возвращает строки из первого набора, которые отсутствуют во втором

## 60. Что такое оконные функции и как они используются?

Оконные функции выполняют вычисления над набором строк, связанных с текущей строкой. Они не сокращают количество строк результата.

Примеры оконных функций:
- ROW_NUMBER() - номер строки
- RANK() - ранг с пропусками
- DENSE_RANK() - плотный ранг без пропусков
- LAG() - доступ к предыдущей строке
- LEAD() - доступ к следующей строке

Пример:
```sql
SELECT name, salary,
       ROW_NUMBER() OVER (ORDER BY salary DESC) AS rank
FROM employees;
```

## 61. Что вы можете сказать о WAL (ведение журнала с опережением записи)?

WAL (Write-Ahead Logging) - это метод обеспечения целостности данных. Принцип заключается в том, что любые изменения в базе данных сначала записываются в журнал (лог), а затем применяются к основным файлам данных.

Преимущества:
- Обеспечение целостности данных
- Восстановление после сбоев
- Улучшение производительности (изменения группируются в один журнал)
- Возможность создания резервных копий "горячим" способом

## 62. Как узнать длину строки с помощью SQL?

Для определения длины строки в SQL используются следующие функции:

**MySQL**:
```sql
SELECT LENGTH(column_name) FROM table_name;
SELECT CHAR_LENGTH(column_name) FROM table_name; -- для юникода
```

**PostgreSQL**:
```sql
SELECT LENGTH(column_name) FROM table_name;
SELECT CHAR_LENGTH(column_name) FROM table_name;
```

**SQL Server**:
```sql
SELECT LEN(column_name) FROM table_name;
```

## 64. В чём различия между OLTP и OLAP?

**OLTP (Online Transaction Processing)**:
- Оптимизирован для транзакций
- Частые небольшие операции чтения/записи
- Нормализованные данные
- Примеры: банковские операции, заказы

**OLAP (Online Analytical Processing)**:
- Оптимизирован для анализа
- Редкие, но сложные запросы
- Денормализованные данные (звезда, снежинка)
- Примеры: отчеты, аналитика, сводные данные

## 65. Что такое партиционирование базы данных?

Партиционирование - это процесс разделения таблицы или индекса на более мелкие, управляемые части без изменения приложения. Каждое разделение хранит часть данных, но таблица остается логически цельной.

Типы партиционирования:
- По диапазону (Range)
- По списку (List)
- По хэшу (Hash)
- По столбцу (Column)

Преимущества:
- Улучшение производительности
- Упрощение обслуживания
- Более эффективные операции резервного копирования

## 66. Что такое шардирование/шардинг базы данных?

Шардинг - это метод горизонтального разделения данных в распределенной базе данных. В отличие от партиционирования, шарды физически распределены по разным серверам.

Характеристики шардинга:
- Распределение данных по разным физическим узлам
- Может улучшить масштабируемость
- Требует стратегии выбора шарда
- Усложняет JOIN-операции между шардами

Преимущества:
- Масштабирование производительности
- Увеличение пропускной способности
- Повышенная доступность

Недостатки:
- Сложность управления транзакциями
- Сложность JOIN-операций
- Балансировка нагрузки между шардами
