## 1. Что такое Docker?

**Ответ:** Docker — это платформа с открытым исходным кодом, которая автоматизирует развертывание, масштабирование и управление приложениями с помощью контейнеризации. Docker позволяет упаковывать приложения и их зависимости в легковесные и переносимые контейнеры, которые могут работать в любой системе с поддержкой Docker.

**Преимущества:**
- Изоляция приложений и зависимостей
- Переносимость между различными средами
- Быстрое развертывание и масштабирование
- Устойчивость к конфликту зависимостей

---

## 2. Опишите составные части архитектуры Docker

**Ответ:** Архитектура Docker включает следующие компоненты:

- **Docker Client** — интерфейс командной строки (CLI), с помощью которого пользователи управляют Docker-демоном
- **Docker Host** — хост, на котором запущен Docker-демон, а также контейнеры, образы, сети и тома
- **Docker Daemon** — фоновый процесс, который слушает Docker API-запросы и управляет Docker-объектами (образами, контейнерами, сетями, томами и т.д.)
- **Docker Registry** — репозиторий для хранения и распространения Docker-образов (например, Docker Hub)
- **Docker Images** — неизменяемые шаблоны для создания контейнеров
- **Docker Containers** — запущенные экземпляры Docker-образов

---

## 3. Какие примеры использования Docker?

**Ответ:** Примеры использования Docker:

- **Разработка и тестирование** — обеспечение одинаковой среды разработки для всех участников команды
- **Непрерывная интеграция и развертывание (CI/CD)** — автоматизация процесса сборки и развертывания приложений
- **Микросервисы** — разбиение приложения на небольшие независимые сервисы, которые могут быть развернуты и масштабированы отдельно
- **Облачные вычисления** — использование контейнеров для эффективного размещения приложений в облаке
- **Кроссплатформенность** — возможность запуска одного и того же приложения на разных операционных системах

## 4. Как определить состояние контейнера Docker?

**Ответ:** Для определения состояния контейнера Docker можно использовать команду:
```bash
docker ps -a
```

Эта команда показывает все контейнеры и их текущее состояние (running, exited, paused и т.д.).

Также можно использовать:
```bash
docker inspect <container_id>
```
для получения подробной информации о контейнере, включая его состояние, конфигурацию, метаданные и т.д.

## 5. Расскажите о наиболее общих командах Docker?

**Ответ:**

- `docker run` — создает и запускает новый контейнер
- `docker ps` — показывает список запущенных контейнеров
- `docker ps -a` — показывает все контейнеры (запущенные и остановленные)
- `docker stop` — останавливает запущенный контейнер
- `docker start` — запускает остановленный контейнер
- `docker rm` — удаляет контейнер
- `docker images` — показывает список всех образов
- `docker rmi` — удаляет образ
- `docker logs` — выводит логи контейнера
- `docker exec` — запускает команду в работающем контейнере
- `docker build` — собирает образ из Dockerfile
- `docker pull` — скачивает образ из репозитория
- `docker push` — загружает образ в репозиторий

## 6. Расскажите о продвинутых командах Docker?

**Ответ:**

- `docker stats` — отображает статистику использования ресурсов (CPU, память и т.д.) для контейнеров
- `docker top` — показывает запущенные процессы в контейнере
- `docker diff` — показывает изменения файловой системы контейнера
- `docker port` — показывает сопоставление портов контейнера
- `docker wait` — блокирует выполнение до завершения работы контейнера
- `docker attach` — подключается к запущенному контейнеру
- `docker commit` — создает образ из измененного контейнера
- `docker export` — экспортирует файловую систему контейнера
- `docker network create` — создает новую сеть
- `docker volume create` — создает новый том

## 7. Управление Docker

**Ответ:** Управление Docker включает:

- Управление контейнерами (запуск, остановка, удаление)
- Управление образами (сборка, тегирование, удаление)
- Управление сетями (создание, подключение, удаление)
- Управление томами (создание, монтирование, удаление)
- Мониторинг и логирование (просмотр статистики, логов)
- Безопасность (ограничение доступа, сканирование образов)
- Оркестрация (использование Docker Compose, Swarm или Kubernetes)

## 8. Что такое виртуализация?

**Ответ:** Виртуализация — это технология, которая позволяет создавать виртуальные версии физических ресурсов, таких как серверы, операционные системы, хранилище данных и сетевые ресурсы. В контексте вычислительных систем виртуализация позволяет запускать несколько виртуальных машин (VM) на одном физическом сервере с помощью гипервизора, который управляет распределением ресурсов между этими VM.

---

## 9. Что такое гипервизор?

**Ответ:** Гипервизор — это программный слой, который создает и управляет виртуальными машинами (VM). Он позволяет нескольким операционным системам работать на одном физическом хосте, изолируя их друг от друга. Существуют два типа гипервизоров:

- **Type 1 (голый металл)** — работает непосредственно на физическом оборудовании (например, VMware vSphere, Microsoft Hyper-V, Xen)
- **Type 2** — работает поверх уже установленной операционной системы (например, Oracle VirtualBox, VMware Workstation)

## 10. Различия контейнеризации и виртуализации?

**Ответ:**

| Характеристика | Виртуализация | Контейнеризация |
|----------------|---------------|------------------|
| **Изоляция** | Полная изоляция с отдельной ОС | Процесс-уровневая изоляция в ядре |
| **Производительность** | Накладные расходы из-за эмуляции ОС | Высокая производительность |
| **Размер** | Больше места из-за ОС | Компактные образы |
| **Запуск** | Медленнее из-за загрузки ОС | Быстрее |
| **Портабельность** | Менее портативные | Высокая портабельность |
| **Безопасность** | Лучшая изоляция | Меньше изоляции, но более гибкое управление |

## 11. Что такое Docker контейнер?

**Ответ:** Docker контейнер — это легковесный, портативный и самодостаточный исполняемый пакет, который включает в себя приложение и все его зависимости (библиотеки, системные инструменты, параметры и т.д.). Контейнер — это запущенный экземпляр Docker-образа, который может быть создан, запущен, остановлен, перемещен или удалён. Контейнеры изолированы друг от друга и от хост-системы.

---

## 12. Как реализовать автоматический перезапуск Docker контейнера?

**Ответ:** Для автоматического перезапуска контейнера используется флаг `--restart` с командой `docker run`:

```bash
docker run --restart=always <image_name>
```

Варианты политики перезапуска:
- `no` — не перезапускать контейнер (по умолчанию)
- `on-failure[:max-retries]` — перезапускать только при сбоях с необязательным ограничением попыток
- `unless-stopped` — перезапускать при сбоях, но не при ручной остановке
- `always` — всегда перезапускать, независимо от причины остановки

---

## 13. Жизненный цикл контейнера

**Ответ:** Жизненный цикл Docker-контейнера включает следующие этапы:

1. **Создание** — контейнер создается из образа
2. **Запуск** — контейнер запускается и начинает выполнение процесса
3. **Работа** — контейнер выполняет задачи
4. **Пауза** — контейнер может быть приостановлен (не все процессы могут быть приостановлены)
5. **Остановка** — контейнер останавливается, процесс завершается
6. **Удаление** — контейнер может быть удален

## 14. Опишите все возможные состояния контейнера Docker

**Ответ:**

- **Created** — контейнер был создан, но еще не запущен
- **Running** — контейнер запущен и работает
- **Paused** — контейнер приостановлен (все процессы в контейнере приостановлены)
- **Restarting** — контейнер находится в процессе перезапуска
- **Removing** — контейнер находится в процессе удаления
- **Exited** — контейнер остановлен после завершения работы
- **Dead** — контейнер не может быть остановлен или удалён

## 15. Что такое Docker Compose?

**Ответ:** Docker Compose — это инструмент для определения и запуска многоконтейнерных Docker-приложений. С помощью Compose вы можете использовать YAML-файл для настройки сервисов вашего приложения, а затем с одной командой создавать и запускать все сервисы, определенные в конфигурации.

Пример команды:
```bash
docker-compose up
```

**Преимущества:**
- Определение всех сервисов в одном файле
- Единая команда для запуска всех сервисов
- Автоматическая настройка сетей между контейнерами

---

## 16. Что такое Dockerfile?

**Ответ:** Dockerfile — это текстовый файл, содержащий последовательность инструкций для создания Docker-образа. Каждая инструкция в Dockerfile создает новый слой в образе. Docker читает инструкции из Dockerfile и выполняет их последовательно для автоматического создания образа.

Пример простого Dockerfile:
```dockerfile
FROM ubuntu:latest
RUN apt-get update
COPY . /app
WORKDIR /app
CMD ["python", "app.py"]
```

## 17. В чем разница между `CMD` и `ENTRYPOINT` в Dockerfile?

**Ответ:**

- **CMD** — предоставляет значения по умолчанию для запуска контейнера. Эти значения могут быть переопределены при запуске контейнера с помощью команды `docker run`.

- **ENTRYPOINT** — конфигурирует контейнер для запуска как исполняемый файл. При использовании ENTRYPOINT команды, переданные в `docker run`, будут добавлены как аргументы к команде ENTRYPOINT.

**Пример:**
```dockerfile
ENTRYPOINT ["python"]
CMD ["app.py"]
```
Запуск: `docker run myimage test.py` выполнит `python test.py`.

---

## 18. Что такое образ Docker?

**Ответ:** Docker-образ — это неизменяемый, исполняемый пакет, который включает в себя все необходимое для запуска приложения: код, библиотеки, среду выполнения, параметры и системные библиотеки. Образы используются для создания контейнеров. Образ Docker состоит из ряда слоев, каждый из которых представляет собой изменение в предыдущем слое.

---

## 19. Что такое Docker Registry?

**Ответ:** Docker Registry — это сервис для хранения и распространения Docker-образов. Docker Hub — это публичный реестр, но организации могут использовать частные реестры. Registry позволяет загружать (push) и скачивать (pull) Docker-образы.

---

## 20. Стандартные сети Docker?

**Ответ:** Docker создает три стандартных сети:

- **Bridge** (по умолчанию) — сеть по умолчанию, используемая при запуске контейнера без указания другой сети
- **Host** — убирает изоляцию сети между контейнером и хостом
- **None** — отключает все сети для контейнера

Команда для просмотра сетей:
```bash
docker network ls
```

## 21. Средства оркестрации

**Ответ:** Средства оркестрации Docker:

- **Docker Swarm** — встроенный инструмент оркестрации Docker для управления кластером Docker-хостов
- **Kubernetes** — популярная платформа для автоматизации развертывания, масштабирования и управления контейнеризированными приложениями
- **Apache Mesos** — альтернативная платформа оркестрации
- **Nomad** — простая и гибкая система оркестрации от HashiCorp

## 22. Что такое Docker Swarm

**Ответ:** Docker Swarm — это встроенный в Docker инструмент оркестрации, который позволяет создавать и управлять кластером Docker-хостов. Swarm включает в себя распределенную систему управления контейнерами, автоматическое балансирование нагрузки, отказоустойчивость и масштабирование.

**Основные компоненты:**
- **Manager nodes** — управляют кластером и оркестрацией
- **Worker nodes** — выполняют контейнеры
- **Services** — определяют задачи, которые должны выполняться в кластере

## 23. Что такое volumes в Docker?

**Ответ:** Volumes (тома) в Docker — это механизм для сохранения данных, создаваемых и используемых Docker-контейнерами. Volumes позволяют сохранять данные независимо от жизненного цикла контейнера. В отличие от связывания каталогов (bind mounts), volumes управляются Docker и обеспечивают лучшую производительность и безопасность.

Типы:
- **Named volumes** — именованные тома, создаваемые Docker
- **Anonymous volumes** — неименованные тома
- **Bind mounts** — монтирование хост-директории или файла

---

## 24. Сохраняются ли данные при остановке контейнера?

**Ответ:** Нет, данные в файловой системе контейнера не сохраняются при его остановке. Когда контейнер удаляется, все его внутренние изменения теряются. Для сохранения данных необходимо использовать **тома (volumes)** или **связывание каталогов (bind mounts)**, которые позволяют хранить данные вне контейнера.

Пример использования тома:
```bash
docker run -v myvolume:/data myimage
```
