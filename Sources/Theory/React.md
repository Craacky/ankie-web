## Основы React

### 1. Что такое алгоритм реконсиляции (reconciliation) в React?

**Ответ:** Алгоритм реконсиляции - это процесс, с помощью которого React обновляет DOM. При изменении состояния или props React создает новый Virtual DOM и сравнивает его с предыдущим Virtual DOM. Затем применяет только те изменения, которые действительно необходимы к реальному DOM, что значительно повышает производительность.

**Принципы работы:**
- **Сравнение деревьев:** React начинает сравнение с корня дерева
- **Сравнение элементов одного типа:** Если элементы одного типа, React сравнивает только атрибуты и содержимое
- **Сравнение списка детей:** Используется ключ (key) для определения добавленных, удаленных или измененных элементов

### 2. Что такое Virtual DOM и краткий принцип работы?

**Ответ:** Virtual DOM - это легковесное представление реального DOM в виде JavaScript-объектов. React использует его для эффективного обновления пользовательского интерфейса.

**Принцип работы:**
1. При изменении состояния создается новое дерево Virtual DOM
2. Сравнивается с предыдущим деревом (diffing)
3. Вычисляются минимальные изменения
4. Применяются изменения к реальному DOM

### 3. Почему сравнение Virtual DOM и реального DOM происходит очень быстро?

**Ответ:** 
- Virtual DOM - это просто JavaScript-объекты, что делает операции с ними быстрыми
- React использует оптимизированный алгоритм diffing (O(n) вместо O(n³))
- Обновления группируются и применяются пакетно
- Используется эвристика "одинаковые типы элементов" для быстрого сравнения

### 4. Что такое элемент? Какие свойства он имеет и особенности?

**Ответ:** Элемент в React - это обычный JavaScript-объект, описывающий узел DOM или компонент. Это минимальная единица React-приложения.

```javascript
const element = {
  type: 'h1',
  props: {
    className: 'greeting',
    children: 'Привет, мир!'
  }
};
```

**Свойства элемента:**
- `type` - тип узла (тег, компонент)
- `props` - свойства элемента
- `key` - уникальный идентификатор (опционально)
- `ref` - ссылка на DOM-элемент (опционально)

### 5. Что такое props?

**Ответ:** Props (свойства) - это объект, содержащий данные, передаваемые в компонент извне. Props являются неизменяемыми (read-only) и передаются только сверху вниз.

```javascript
function Welcome(props) {
  return <h1>Привет, {props.name}</h1>;
}

// Использование
<Welcome name="Алиса" />
```

### 6. Что такое children?

**Ответ:** `children` - это специальное свойство props, которое позволяет передавать дочерние элементы внутрь компонента.

```javascript
function Card(props) {
  return (
    <div className="card">
      {props.children}
    </div>
  );
}

// Использование
<Card>
  <h2>Заголовок</h2>
  <p>Содержимое</p>
</Card>
```

### 7. В чем разница между рендерингом и монтированием компонента?

**Ответ:**
- **Монтирование** - это процесс первоначального создания и вставки компонента в DOM
- **Рендеринг** - это процесс обновления представления компонента в ответ на изменения состояния или props

**Этапы монтирования:**
1. constructor()
2. render()
3. componentDidMount()

**Этапы рендеринга (обновления):**
1. getDerivedStateFromProps()
2. shouldComponentUpdate()
3. render()
4. componentDidUpdate()

### 8. В результате чего появляется Virtual DOM?

**Ответ:** Virtual DOM появился как решение проблемы производительности при частых обновлениях DOM. Прямые манипуляции с DOM дороги, поэтому React создает виртуальное представление для эффективного сравнения и минимальных изменений.

### 9. Опишите процесс рендеринга и почему он происходит так быстро?

**Ответ:** 
- React создает Virtual DOM дерево
- При изменениях React сравнивает новое и старое деревья
- Используется оптимизированный алгоритм для вычисления минимальных изменений
- Применяются только необходимые изменения к реальному DOM
- Все обновления группируются и применяются пакетно

### 10. Почему происходит рендеринг компонента (4 пункта)?

**Ответ:**
1. **Изменение состояния (state)** - при вызове setState или изменении React-ссылки
2. **Изменение props** - когда родительский компонент передает новые props
3. **Изменение контекста (context)** - при изменении значений, передаваемых через Context API
4. **Вызов forceUpdate** - принудительное обновление (редко используется)

### 11. Какие преимущества и недостатки у React?

**Преимущества:**
- **Компонентный подход** - переиспользуемость кода
- **Высокая производительность** - благодаря Virtual DOM
- **Огромная экосистема** - множество библиотек и инструментов
- **Однонаправленный поток данных** - легче отлаживать
- **Поддержка JSX** - удобный способ описания UI

**Недостатки:**
- **Крутая кривая обучения** - множество концепций для понимания
- **Частые обновления** - сложно следить за изменениями
- **Объемный фреймворк** - много зависимостей
- **SEO сложности** - проблемы с индексацией без SSR

## Компоненты и жизненный цикл

### 12. Как выбрать между классовыми и функциональными компонентами?

**Ответ:**
- **Функциональные компоненты:** Используйте для простых компонентов, которые не требуют состояния или методов жизненного цикла (или используйте хуки)
- **Классовые компоненты:** Ранее использовались для компонентов с состоянием и методами жизненного цикла, но теперь хуки делают функциональные компоненты предпочитаемыми

**Совет:** С версии React 16.8 и появлением хуков, функциональные компоненты стали предпочтительным выбором.

### 13. Что такое.refs?

**Ответ:** Refs (ссылки) - это способ получения прямого доступа к DOM-элементу или экземпляру компонента. Используются для:
- Управления фокусом
- Вызовов императивных методов
- Интеграции со сторонними библиотеками

```javascript
function TextInput() {
  const inputRef = useRef();

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <div>
      <input ref={inputRef} type="text" />
      <button onClick={focusInput}>Фокус на инпут</button>
    </div>
  );
}
```

### 14. Контролируемые компоненты vs неконтролируемые компоненты?

**Ответ:**

**Контролируемые компоненты:**
- Значение элемента формы контролируется React-состоянием
- Любое изменение значения ввода должно обрабатываться функцией handleChange
- Данные всегда синхронизированы с состоянием

```javascript
function ControlledInput() {
  const [value, setValue] = useState('');

  return (
    <input
      value={value}
      onChange={(e) => setValue(e.target.value)}
    />
  );
}
```

**Неконтролируемые компоненты:**
- Значение элемента формы контролируется DOM
- Используют ref для получения значений
- Ближе к обычному HTML

```javascript
function UncontrolledInput() {
  const inputRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log(inputRef.current.value);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={inputRef} type="text" />
      <button>Отправить</button>
    </form>
  );
}
```

### 15. Этапы жизненного цикла компонента?

**Ответ:**
**Фаза монтирования (Mounting):**
- `constructor()`
- `getDerivedStateFromProps()`
- `render()`
- `componentDidMount()`

**Фаза обновления (Updating):**
- `getDerivedStateFromProps()`
- `shouldComponentUpdate()`
- `render()`
- `getSnapshotBeforeUpdate()`
- `componentDidUpdate()`

**Фаза размонтирования (Unmounting):**
- `componentWillUnmount()`

### 16. Что такое порталы в React?

**Ответ:** Portal - это способ рендеринга дочерних элементов в DOM-узел, который находится за пределами иерархии DOM родительского компонента.

```javascript
import { createPortal } from 'react-dom';

function Modal({ children, isOpen }) {
  if (!isOpen) return null;

  return createPortal(
    <div className="modal">
      {children}
    </div>,
    document.getElementById('modal-root')
  );
}
```

### 17. Что можно реализовать с помощью классовых, чего нельзя в функциональных?

**Ответ:** 
С появлением хуков (React 16.8) функциональные компоненты получили возможность использовать состояние и методы жизненного цикла. Ранее классовые компоненты имели следующие возможности:
- this.state и this.setState для управления состоянием
- Методы жизненного цикла (componentDidMount, componentDidUpdate и др.)
- Ссылки через createRef

Сейчас функциональные компоненты с хуками являются предпочтительным выбором.

### 18. Отличие pure-компонента от обычного в классовых компонентах?

**Ответ:**
- **Обычный компонент:** Обновляется при любом изменении props или state
- **PureComponent:** Выполняет поверхностное сравнение props и state, обновляется только при обнаружении изменений

```javascript
// Обычный компонент
class RegularComponent extends React.Component {
  render() {
    return <div>{this.props.value}</div>;
  }
}

// Pure компонент
class PureComponentExample extends React.PureComponent {
  render() {
    return <div>{this.props.value}</div>;
  }
}
```

## Хуки

### 19. React хуки - перечислите и расскажите?

**Ответ:** 
- **useState** - для управления состоянием
- **useEffect** - для побочных эффектов
- **useContext** - для доступа к контексту
- **useReducer** - альтернатива useState для сложного состояния
- **useCallback** - мемоизация функций
- **useMemo** - мемоизация вычислений
- **useRef** - доступ к DOM-элементам
- **useLayoutEffect** - выполнение эффектов перед рендерингом
- **useImperativeHandle** - кастомизация ref
- **useDebugValue** - отладочная информация

### 20. useState - особенности?

**Ответ:**
- Обновляет состояние асинхронно
- Объединяет вызовы setState в одном цикле обновления
- Не мутирует предыдущее состояние, а заменяет его полностью

```javascript
function Counter() {
  const [count, setCount] = useState(0);

  const increment = () => {
    // Правильный способ
    setCount(prevCount => prevCount + 1);
  };

  return <button onClick={increment}>{count}</button>;
}
```

### 21. useEffect - особенности?

**Ответ:**
- Выполняет побочные эффекты
- Может возвращать функцию очистки
- Выполняется после коммита изменений в DOM
- Может принимать массив зависимостей

```javascript
function Component({ userId }) {
  useEffect(() => {
    const loadUser = async () => {
      const response = await fetch(`/api/users/${userId}`);
      const userData = await response.json();
      // Обработка данных
    };

    loadUser();

    // Функция очистки
    return () => {
      // Очистка ресурсов
    };
  }, [userId]); // Зависимости
}
```

### 22. Правила ограничения использования хуков?

**Ответ:**
1. **Вызывайте хуки только на верхнем уровне** - не используйте хуки внутри условий, циклов или вложенных функций
2. **Вызывайте хуки только из React-функций** - не вызывайте из обычных JavaScript-функций
3. **Следите за массивом зависимостей** - убедитесь, что все зависимости добавлены в массив

### 23. useCallback для чего нужен и когда использовать?

**Ответ:** `useCallback` мемоизирует функцию, предотвращая её повторное создание при каждом рендеринге. Используется для оптимизации производительности.

```javascript
function Parent({ data }) {
  const handleClick = useCallback((item) => {
    console.log(item);
  }, []); // Функция не будет пересоздаваться

  return (
    <div>
      {data.map(item => (
        <Child key={item.id} onClick={handleClick} />
      ))}
    </div>
  );
}
```

### 24. В useEffect есть колбек, колбек возвращает функцию. Что за функция и какие есть кейсы применения?

**Ответ:** Функция, возвращаемая из useEffect, является функцией очистки. Она выполняется перед размонтированием компонента или перед повторным выполнением эффекта.

**Кейсы применения:**
- Очистка таймеров
- Отписка от событий
- Отмена сетевых запросов
- Очистка подписок

```javascript
useEffect(() => {
  const timer = setTimeout(() => {
    console.log('Таймер сработал');
  }, 1000);

  // Функция очистки
  return () => {
    clearTimeout(timer); // Очистка таймера
  };
}, []);
```

### 25. В чем отличие useEffect от useLayoutEffect?

**Ответ:**
- **useEffect** выполняется асинхронно после рендеринга, не блокирует отображение
- **useLayoutEffect** выполняется синхронно перед рендерингом, может блокировать отображение

Используйте `useLayoutEffect` когда нужно избежать мерцания или выполнить синхронные изменения DOM.

### 26. Может ли быть асинхронная setup функция в useEffect? Почему? Как выполнить асинхронные действия внутри него?

**Ответ:** 
- **Нельзя** напрямую использовать async в useEffect
- Причина: async функция возвращает Promise, а useEffect ожидает функцию очистки

**Решение:**
```javascript
useEffect(() => {
  async function fetchData() {
    const response = await fetch('/api/data');
    const data = await response.json();
    // Обработка данных
  }

  fetchData();
}, []);

// Или через IIFE
useEffect(() => {
  (async () => {
    const response = await fetch('/api/data');
    const data = await response.json();
  })();
}, []);
```

### 27. Как отследить демонтирование функционального компонента?

**Ответ:** Воспользуйтесь функцией очистки в useEffect. Если компонент демонтируется, функция очистки будет вызвана:

```javascript
function Component() {
  useEffect(() => {
    let isMounted = true;

    fetch('/api/data')
      .then(response => response.json())
      .then(data => {
        if (isMounted) {
          // Обновляем состояние только если компонент еще смонтирован
        }
      });

    return () => {
      isMounted = false; // Отслеживаем демонтирование
    };
  }, []);

  return <div>Компонент</div>;
}
```

### 28. Как реализовать однократное выполнение операции при начальном рендеринге?

**Ответ:** Используйте useEffect с пустым массивом зависимостей:

```javascript
function Component() {
  useEffect(() => {
    console.log('Выполняется только при монтировании');
    // Выполнение однократной операции
  }, []); // Пустой массив - выполнится только один раз

  return <div>Компонент</div>;
}
```

## Управление состоянием

### 29. Какие подходы к управлению состоянием в React существуют?

**Ответ:**
1. **Локальное состояние** - useState, useReducer
2. **Контекст** - useContext для передачи данных глубоко в дереве
3. **Redux** - централизованное управление состоянием
4. **MobX** - реактивное управление состоянием
5. **Recoil** - от Facebook, для сложных состояний
6. **Zustand** - легковесный менеджер состояния

### 30. Ключевые принципы Redux?

**Ответ:**
- **Единственный источник истины:** Вся информация о состоянии приложения хранится в одном объекте store
- **Состояние доступно только для чтения:** Для изменения состояния нужно отправить action
- **Изменения происходят через чистые функции:** Редьюсеры - чистые функции, определяющие как изменяется состояние

### 31. Что такое редьюсер?

**Ответ:** Редьюсер (reducer) - это чистая функция, которая принимает предыдущее состояние и экшен, и возвращает новое состояние. Он определяет как изменяется состояние приложение.

```javascript
function counterReducer(state = 0, action) {
  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    case 'DECREMENT':
      return state - 1;
    default:
      return state;
  }
}
```

### 32. Как редьюсер связан с обновлением состояния в Redux?

**Ответ:** 
1. Компонент диспетчерит action (объект с типом и полезной нагрузкой)
2. Redux передает action в редьюсер вместе с текущим состоянием
3. Редьюсер возвращает новое состояние на основе action.type
4. Redux обновляет store и уведомляет подписчиков

### 33. Разница между React State и Redux State?

**Ответ:**
- **React State:** Локальное состояние компонента, обновляется через setState/useState
- **Redux State:** Глобальное состояние приложения, обновляется через редьюсеры и actions

React State подходит для локальных данных, Redux State для данных, которые используются во многих частях приложения.

### 34. Контекст vs Redux?

**Ответ:**
- **Контекст:** Простое решение для передачи данных по дереву компонентов без проп-дриллинга
- **Redux:** Полноценное решение для управления сложным состоянием с middleware, DevTools и т.д.

Используйте контекст для небольших приложений, Redux для сложных с большим объемом состояния.

## Оптимизация и производительность

### 35. Зачем нужны useMemo и useCallback?

**Ответ:**
- **useMemo** - мемоизирует вычисления, предотвращая их повторное выполнение при каждом рендеринге
- **useCallback** - мемоизирует функции, предотвращая их пересоздание

```javascript
// useMemo для вычислений
const expensiveValue = useMemo(() => {
  return data.filter(item => item.active).map(item => item.name);
}, [data]);

// useCallback для функций
const handleClick = useCallback((id) => {
  console.log(id);
}, []);
```

### 36. Разница между memo и useMemo?

**Ответ:**
- **React.memo** - мемоизация для компонентов (аналог PureComponent)
- **useMemo** - мемоизация для значений/вычислений внутри компонента

```javascript
// memo для компонента
const MyComponent = React.memo(({ data }) => {
  return <div>{data}</div>;
});

// useMemo для значения
function Parent({ items }) {
  const expensiveValue = useMemo(() => {
    return items.filter(item => item.active);
  }, [items]);

  return <MyComponent data={expensiveValue} />;
}
```

### 37. memo vs useMemo vs useCallback?

**Ответ:**
- **memo:** Мемоизация компонента, предотвращает рендеринг при одинаковых props
- **useMemo:** Мемоизация вычислений, предотвращает повторные вычисления
- **useCallback:** Мемоизация функций, предотвращает пересоздание функций

### 38. Какие есть кейсы использования useRef?

**Ответ:**
- Доступ к DOM-элементам
- Хранение изменяемых значений (не вызывает перерендер)
- Сохранение предыдущих значений props/state

```javascript
function TextInput() {
  const inputRef = useRef();

  const focusInput = () => {
    inputRef.current.focus();
  };

  return <input ref={inputRef} />;
}
```

### 39. Для чего нужен key в React?

**Ответ:** Key помогает React определить, какие элементы были изменены, добавлены или удалены. Ключ должен быть уникальным и стабильным для каждого элемента в списке.

```javascript
function List({ items }) {
  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>{item.name}</li> // Уникальный ключ
      ))}
    </ul>
  );
}
```

### 40. Как создать компонент высшего порядка (HOC)?

### 41. Как реализовать двустороннее связывание данных в неуправляемых компонентах?

**Ответ:** В неуправляемых компонентах двустороннее связывание реализуется через refs:

```javascript
function UncontrolledForm() {
  const inputRef = useRef();

  const handleSubmit = (e) => {
    e.preventDefault();
    // Чтение значения из DOM
    console.log(inputRef.current.value);
  };

  const updateValue = (newValue) => {
    // Установка значения в DOM
    inputRef.current.value = newValue;
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={inputRef} type="text" />
      <button type="submit">Отправить</button>
    </form>
  );
}
```

### 42. Что происходит в этом коде? Где тут двустороннее связывание, управляемый инпут, неуправляемый?

**Ответ:** Без конкретного кода сложно дать точный ответ, но в общем:

- **Управляемый инпут:** Значение контролируется React-состоянием через value и onChange
- **Неуправляемый инпут:** Значение контролируется DOM через ref
- **Двустороннее связывание:** Управление значением через состояние и обновление состояния при изменении

## Лучшие практики

### 43. На что обращаете внимание при рефакторинге и разработке?

**Ответ:**
- **Читаемость кода** - понятные имена переменных и функций
- **Разделение ответственности** - компоненты должны выполнять одну задачу
- **Консистентность** - единый стиль кодирования
- **Производительность** - избегать ненужных рендеров
- **Тестируемость** - писать легко тестируемый код
- **Архитектура** - правильно организовывать структуру проекта

### 44. Что такое проп-дриллинг?

**Ответ:** Проп-дриллинг (prop drilling) - это передача данных через несколько уровней компонентов, когда родитель получает данные, которые нужны только глубоко вложенному потомку.

**Решения:**
- Использование Context API
- Компоненты высшего порядка
- Redux или другие менеджеры состояния
- Реструктуризация дерева компонентов

### 45. Зачем вообще нужны хуки и как они появились?

**Ответ:** Хуки были введены для решения следующих проблем:
- Повторение логики между компонентами
- Невозможность переиспользовать состояние и побочные эффекты в функциональных компонентах
- Сложность понимания классовых компонентов (this, жизненный цикл)
- Комплексность тестирования

Хуки позволяют использовать состояние и другие возможности React в функциональных компонентах.

### 46. Что будет если у useEffect убрать массив зависимостей?

**Ответ:** Если убрать массив зависимостей из useEffect, функция будет выполняться после каждого рендеринга компонента, что может привести к:
- Бесконечному циклу обновлений
- Плохой производительности
- Нежелательным побочным эффектам

### 47. Что такое Context API и где используется?

**Ответ:** Context API - это способ передачи данных через дерево компонентов без проп-дриллинга. Используется для:
- Темизации
- Аутентификации
- Локализации
- Общего состояния приложения

```javascript
const ThemeContext = createContext();

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <ChildComponent />
    </ThemeContext.Provider>
  );
}
```
